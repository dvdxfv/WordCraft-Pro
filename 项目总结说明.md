## 项目架构与交互机制

### 整体架构

WordCraft Pro 采用 **pywebview + HTML/JS/CSS + Python 后端** 的三层架构设计，结合了 Web 前端的灵活性和桌面应用的功能性。

```
┌─────────────────────────────────────────────────────────┐
│                    WordCraft Pro                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────────┐    pywebview 桥接     ┌─────────────┐│
│  │  前端 (Web)   │ ←──────────────────→ │  后端 (Python)│
│  │              │                        │             ││
│  │  HTML/CSS/JS │                        │  app.py     ││
│  │  界面渲染     │                        │  文件解析    ││
│  │  用户交互     │                        │  质量检查    ││
│  │  状态管理     │                        │  AI 调用     ││
│  └──────────────┘                        └─────────────┘│
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 前端与后端交互机制

#### 1. pywebview 桥接

前端通过 `window.pywebview.api` 调用 Python 后端方法，实现双向通信：

**前端调用后端：**
```javascript
// 打开文件
const filePath = await window.pywebview.api.openFile();

// 运行质量检查
const result = await window.pywebview.api.runQA(content, categories);

// 运行交叉引用检查
const xrefResult = await window.pywebview.api.runXRef(content);

// 调用 AI 分析
const aiResult = await window.pywebview.api.callAI(systemPrompt, userMessage, config);
```

**后端暴露给前端：**
```python
# app.py 中的 Bridge 类
class Bridge:
    def openFile(self) -> str: ...           # 打开文件对话框
    def saveDocument(self, content, title) -> str: ...  # 保存文档
    def runQA(self, content, categories) -> str: ...    # 质量检查
    def runXRef(self, content) -> str: ...                # 交叉引用检查
    def callAI(self, system, user, config) -> str: ...    # AI 调用
    def exportDocument(self, content) -> str: ...         # 导出文档
```

#### 2. 用户登录交互

**登录流程：**
1. 用户打开应用 → 显示登录界面
2. 输入邮箱/手机号 + 密码 → 调用 Supabase Auth API
3. 验证通过 → 获取 JWT Token → 存储到本地
4. Token 有效期内自动登录 → 加载用户数据和模板配置

**技术实现：**
- **Supabase Auth**：提供邮箱/手机号登录、密码重置、Session 管理
- **本地存储**：Token 和刷新 Token 存储在 `localStorage`
- **自动续期**：Token 过期前自动刷新，保持登录状态
- **离线模式**：核心功能不依赖网络，登录仅用于同步模板和用量统计

**登录后的数据同步：**
- 用户模板配置（排版规则、检查规则）从云端加载
- Token 用量从云端同步
- 最近使用的文档列表同步

**核心文件：** `web/index.html`（167KB，包含所有 CSS 和 JS）

**界面布局：**
```
┌─────────────────────────────────────────────────────────┐
│  工具栏：Logo | 打开 | 保存 | 排版 | 质量检查 | 交叉引用 | 导出 │

│          │                            │                 │
│  文件树  │       文档预览/编辑区       │    右侧面板     │

│  状态栏：状态 | 字数统计 | Token 用量 | 版本号              │
```

**前端核心模块：**
- **文件管理**：多标签页、文件树、拖拽上传
- **文档预览**：A4 纸张模拟、富文本编辑、高亮标注
- **右侧面板**：排版设置、质量检查列表、交叉引用管理
- **AI 集成**：排版规则解析、QA 深度分析

### 后端架构

**核心文件：** `app.py`（pywebview 应用入口）

**核心模块：**
- **文件解析**：docx/txt/md/pdf 文件解析器
- **排版引擎**：根据规则或 AI 解析结果应用排版样式
- **质量检查**：错别字、一致性、逻辑、LLM 增强检测
- **交叉引用**：扫描、匹配、高亮、自动插入
- **AI 客户端**：调用豆包 API 进行智能分析

## 核心功能模块

### 1. 文件管理

- 支持 `.docx`、`.txt`、`.md`、`.pdf` 格式
- 多标签页管理，同时编辑多个文档
- 文件树展示、拖拽添加
- 自动保存（30 秒延迟）
- 修改状态追踪、关闭前确认

### 2. 智能排版

**自定义排版规则：**
- 页面设置（纸张大小、页边距）
- 标题样式（一至三级标题的字体、字号、对齐、行距）
- 正文样式（中/西文字体、字号、首行缩进、行距、对齐）
- 表格样式（表题、表体字体字号、边框、对齐）
- 图题样式（图号格式、字体字号、位置）

**AI 智能排版：**
- 上传模板文件（`.docx`），自动提取样式
- 自然语言描述排版需求，AI 解析为结构化规则
- 预设模板：学术论文、政府公文、实验报告

### 3. 质量检查

**检查类型：**
- **错别字检查**：常见错别字、"的地得"误用、成语错字
- **数据一致性**：数字格式、日期格式、名称统一
- **格式检查**：标题层级、行距、对齐、段落间距
- **LLM 增强检测**：语法错误、逻辑问题、表达优化
- **交叉引用**：悬空引用、未引用目标检测

**检查结果展示：**
- 文档内高亮标注（红色=错误、橙色=警告、蓝色=提示）
- 悬停提示框：问题说明 + 修改建议
- 右侧面板列表：筛选、排序、定位
- 一键接受/拒绝、批量操作

### 4. 交叉引用管理

- 自动扫描文档中的引用目标（图、表、公式、章节、参考文献）
- 匹配引用与目标，检测悬空引用和未引用目标
- 可视化展示引用关系
- 点击采纳自动插入交叉引用

### 5. 文档导出

- 导出为 `.docx` 格式（保留排版样式）
- 导出为 HTML 格式
- （待实现）PDF 导出

## 解决的核心问题

WordCraft Pro 旨在解决 Word 文档排版和校对中的以下痛点问题：

### 问题 1：排版规则不统一，手动调整耗时

**场景描述：**
用户需要根据特定的排版规则（如学术论文规范、政府公文格式）对文档进行排版。手动调整每个标题、段落、表格的字体、字号、行距、对齐方式非常耗时，且容易遗漏。

**解决方案：**
- 用户设定或上传排版规则模板
- 一键应用排版规则，自动调整全文档样式
- AI 智能解析自然语言描述的排版需求

### 问题 2：质量检查发现格式不一致

**场景描述：**
长文档中，相同级别的元素格式不一致，例如：
- 前文的三级标题使用**仿宋**，此处的三级标题却用了**楷体**
- 表格中的文字设定为**宋体十号字**，但发现仍有部分单元格文字是**11 号字**
- 有些段落的行距没有统一（1.5 倍行距 vs 单倍行距）
- 段落中间存在不应有的**空白行**

**解决方案：**
- 质量检查引擎扫描文档，对比实际样式与规则要求
- 高亮标注格式不一致的地方
- 提供修改建议，一键接受修复

### 问题 3：标点符号格式混乱

**场景描述：**
文档中混用中文和西文标点：
- 括号有的是中文格式 `（）`，有的是西文格式 `()`
- 前引号有的是中文格式 `"`，有的是西文格式 `"`

**解决方案：**
- 质量检查识别标点符号格式不一致问题
- 根据文档语言环境推荐统一格式
- 一键替换修正

### 问题 4：错别字和用词错误

**场景描述：**
- 文档中存在常见错别字（如"帐号"应为"账号"、"按装"应为"安装"）
- "的地得"误用
- 成语错字（如"穿流不息"应为"川流不息"）

**解决方案：**
- 规则引擎扫描常见错别字词典
- LLM 增强检测上下文相关的用词错误
- 高亮标注，提供修改建议

### 问题 5：数值前后不统一

**场景描述：**
- 前文写"项目总投入 100 万元"，后文写"项目总投资 1000000 元"
- 同一数据用不同格式表示（"50%" vs "百分之五十"）
- 日期格式不一致（"2024 年 1 月 1 日" vs "2024-01-01"）

**解决方案：**
- 数据一致性检查引擎提取所有数字、日期、名称
- 检测同一数值的不同表示方式
- 提供统一格式建议

### 问题 6：每个人的检查需求不同

**场景描述：**
不同用户、不同文档类型需要的质量检查规则不同：
- 学术论文关注引用格式、图表编号
- 政府公文关注格式规范、用词准确
- 商业报告关注数据一致性、逻辑完整性

**解决方案：**
- 检查规则可动态配置
- 根据上传的模板或用户自定义规则进行检查
- 支持开启/关闭特定检查项

## 交叉引用功能

### 功能描述

交叉引用是长文档（如论文、报告、书籍）中的重要功能。WordCraft Pro 提供完整的交叉引用管理：

### 自动扫描

程序自动扫描文档中的引用目标：
- **图表**：图 1-1、图 2-3、表 1-1、表 3-2
- **公式**：公式 (1)、公式 (2-1)
- **章节**：第一章、第二节、1.1 小节
- **参考文献**：[1]、[2]、[3]

### 检测问题

**悬空引用：**
- 文档中提到"如图 1-1 所示"，但文档中没有图 1-1
- 高亮标注，提示引用目标不存在

**未引用目标：**
- 文档中有"表 2-1"，但全文没有地方引用它
- 提示用户可以删除或补充引用

### 自动插入交叉引用

**操作流程：**
1. 运行交叉引用检查
2. 扫描发现应该有交叉引用但没有的位置（高亮标注）
3. 用户点击高亮位置
4. 弹出提示框，显示可用的引用目标列表
5. 用户选择目标，点击"采纳"
6. 程序自动插入格式正确的交叉引用文本

**示例：**
```
原文：...结果如图所示...（"如图"后高亮）
用户点击 → 选择"图 3-2 实验结果对比" → 点击采纳
结果：...结果如图 3-2 所示...
```

## 打包与部署方案

### 开发完成后：打包为桌面程序

项目完成后，需要将应用打包成独立的桌面程序，供客户下载和使用。

### 打包技术方案

**推荐方案：PyInstaller + pywebview**

```
打包后结构：
WordCraft-Pro-Setup/
├── WordCraft-Pro.exe          # 主程序（含 Python 运行时）
├── web/
│   └── index.html             # 前端界面（不打包，保持可更新）
├── config.yaml                # 配置文件（用户可修改）
├── templates/                 # 预设排版模板
├── core/                      # 核心模块（可选打包或源码分发）
├── requirements.txt           # 依赖清单
└── README.md                  # 使用说明
```

**打包步骤：**
1. 安装 PyInstaller：`pip install pyinstaller`
2. 创建打包配置：`pyinstaller wordcraft-pro.spec`
3. 打包命令：`pyinstaller --onefile --windowed app.py`
4. 打包产物在 `dist/` 目录

**打包配置示例（.spec 文件）：**
```python
a = Analysis(
    ['app.py'],
    pathex=['.'],
    datas=[
        ('web/index.html', 'web'),
        ('config.yaml', '.'),
        ('templates/', 'templates'),
    ],
    ...
)
```

### 客户使用流程

**下载安装：**
1. 客户从官网/下载链接获取安装包
2. 解压到本地目录
3. 双击 `WordCraft-Pro.exe` 启动
4. **无需任何配置，开箱即用！**

**API Key 管理：**
- 豆包 API Key 已内置在后端代码中
- 客户无需手动填写或配置
- 所有 AI 功能（智能排版、QA 深度分析）直接可用
- 开发者通过更新程序版本来轮换 API Key

**联网功能：**
- **AI 功能**：调用豆包 API 需要联网（智能排版、QA 深度分析）
- **质量检查**：规则引擎检测无需联网，LLM 增强检测需要联网
- **登录同步**：Supabase 云服务需要联网（可选功能）
- **离线模式**：基础排版、规则检查、文件管理可离线使用

**（可选）用户登录：**
- 如需云端同步模板和用量统计，可注册 Supabase 账号
- 核心功能不依赖登录，离线也可正常使用

### 分发方式建议

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **绿色版** | 解压即用，无需安装 | 临时测试、快速体验 |
| **安装包** | NSIS/Inno Setup 制作安装程序 | 正式客户分发 |
| **自动更新** | 内置版本检测和更新功能 | 持续迭代维护 |

## 快速开始指南

### 环境要求

- Python 3.8+
- Windows 10/11 或 macOS 10.15+

### 安装依赖

```bash
cd wordcraft-pro
pip install -r requirements.txt
```

### 运行应用

```bash
python app.py
```

### 配置文件

编辑 `config.yaml` 配置 AI API 密钥：

```yaml
llm:
  api:
    provider: "doubao"
    api_key: "your-api-key"
    base_url: "https://ark.cn-beijing.volces.com/api/v3"
    model: "doubao-seed-1-6-251015"
```
