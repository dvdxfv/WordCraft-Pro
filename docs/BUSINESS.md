## 1. 执行摘要

### 我们卖什么、卖给谁、为什么现在可行

**卖什么**：面向中文办公/学术场景的智能文档检查与格式合规 SaaS 工具——规则层（免费）+ AI深度检查（付费），帮用户在提交前发现格式错误、错别字、逻辑漏洞、交叉引用断裂。

**卖给谁（优先级排序）**：
1. 撰写毕业论文/课题报告的研究生与本科生（冷启动种子）
2. 输出研究报告的科研团队与咨询团队（增长主力）
3. 需要规范化文件输出的政府部门/高校机构（长期锚点）

**为什么现在可行**：
- 国内无直接对标产品，Word 内置检查不懂中文学术规范；
- DeepSeek V4 成本极低（单次检查约 ¥0.0013），SaaS 毛利率接近 99%；
- 毕业季（3-6月、9-12月）形成天然流量窗口，获客成本趋近 ¥0；
- 功能已完整可用（58 个测试通过），无需等待开发。

### 80/20 战略一句话定义

> **80% 精力做 SaaS 订阅（长期复购引擎），20% 精力做按次服务与定制（短期现金流补充）；论文季是冷启动杠杆，不是主业。**

#### 计费方式

| 套餐 | 月付 | 年付 | 席位 |

> **首发优惠（冷启动期）**：Pro 首月 ¥6.9（后续 ¥9.9），年付折上折 ¥69（限前100名）。

#### 防止免费用户吃掉成本的机制

免费版 AI 调用成本极低（3次/月 × ¥0.0013 = ¥0.004），主要管控点不是成本，而是**转化漏斗**：

1. **功能门槛**：免费版 AI 配额用尽后，点击相关按钮直接弹出升级 CTA，不允许绕过。
2. **文件大小限制**：免费版单文件 ≤ 5MB；Pro ≤ 30MB；Team ≤ 100MB。超限文件仍可上传但 AI 检查被限流。
3. **频率限制**：免费版规则检查 10次/天（防爬），不影响正常使用。
4. **账号指纹**：注册时记录设备 ID + 邮箱域名，防止批量注册薅免费额度。

|------|------|---------|
| 单次 AI 深度检查（含报告） | ¥19.9/次 | 不想订阅、急用一次 |

| 格式规范模板提取（从 PDF/Word 提取规则） | ¥39/次 | 有规范文件但懒得手动配置 |

> 按次服务是 SaaS 的转化前置，不是独立主营——买3次单次的用户强推年付性价比对比。

#### 定制服务（中大客户现金流）

| 服务 | 定价区间 | 交付物 |
|------|----------|--------|
| 机构格式规范定制 | ¥500–2000/套 | 针对特定期刊/学校/机构的规则库配置 |
| 私有部署方案 | ¥5000–20000/年 | 本地化部署 + 维护 SLA |
| 批量文档审核报告 | ¥200–800/批 | 团队多份文档的格式合规汇总报告 |

**定制服务反哺 SaaS 的机制**：
- 每个定制规则库沉淀为平台内置模板（匿名化后），丰富免费版内容；
- 机构客户案例转化为内容营销素材（征得同意后）；
- 定制过程中暴露的高频需求直接进入产品 backlog。

**红线**：不接受纯外包（如帮客户人工改稿、设计排版）。定制仅限规则库配置和部署，不消耗产品团队核心精力超过 20%。

## 6. 财务与单位经济模型

### 核心指标定义

| 指标 | 计算方式 | 当前估算 |
|------|----------|---------|
| ARPU | 月均收入 / 付费用户数 | ¥9.9（个人）/ ¥9.8（团队席位均摊） |
| CAC | 获客总成本 / 新付费用户数 | ¥0（冷启动期）→ ¥5–20（内容营销期） |
| LTV | ARPU × 平均付费月数 | ¥9.9 × 5 = ¥49.5（保守）/ ¥79（年付） |

## 8. 风险与对策

| 风险 | 概率 | 影响 | 预案 |
|------|------|------|------|
| **AI误报率高导致用户不信任** | 高 | 高 | 规则层免费兜底（准确率高）；AI层标注置信度；允许用户一键反馈误报并积累数据迭代 |
| **DeepSeek API 波动/停服** | 中 | 中 | 已实现双渠道回退（Supabase Edge + 直连）；预备 GLM/Qwen 接口切换方案（1天工作量） |
| **毕业季结束后用户大量流失** | 高 | 中 | Q2 起主动拓展非论文场景（实习报告、科研中期报告、申请书）；年付锁定用户 |
| **同质化竞争（大厂入场）** | 中 | 高 | 深挖中文学术规范细节（大厂不愿精耕），积累用户规则库数据形成护城河；快速迭代 |
| **定制服务失控吃掉产品精力** | 中 | 高 | 明确红线：定制只做规则库配置，不做人工改稿；每月复盘定制占比，超 20% 立即停接新单 |
| **Supabase 免费层暂停**（7天不活跃） | 中 | 低 | 配置定时 ping 脚本保活；商业化后按需升级付费层（$25/月） |
| **2-3人团队交付过载** | 高 | 中 | 按季度 OKR 聚焦单一优先级；禁止同时推进超过2个新功能；每月评估精力分配 |

## 9. 决策清单

### 本周必须做的10件事

1. 在准毕业生同学群发送产品链接 + 简短使用教程（截止日：今天）
2. 确认赞赏码/转账收款方式，写清楚激活流程（10分钟内可完成）
3. 发布第1篇知乎内容（标题：论文格式提交前必查清单）
4. 建立付费用户微信群（即使目前0人）
5. 记录前5个使用反馈（好的/坏的都要，截止本周末）
6. 确认 Supabase 保活脚本已运行
7. 整理1个"成功案例"（哪怕是内部测试文档），写成可分享的截图
8. 在产品内加入"联系我们"入口（微信/邮件），降低用户反馈门槛
9. 统计当前功能列表，确认哪些 Pro 功能已加 Feature Lock（未加的本周补上）
10. 确定第1次"论文格式急诊"服务的定价和交付承诺，写成可转发的一段话

### 本月验证的5个商业假设

| 假设 | 验证方法 | 判定标准 |
|------|----------|---------|
| 毕业生愿意为格式检查付费 | 手工收款，推给 50 人 | ≥5 人付款 = 假设成立 |
| ¥9.9/月 不是价格阻碍 | 对拒绝用户追问原因 | <30% 拒绝原因是"太贵" |
| 规则层检查结果足够准确 | 收集5份真实文档的用户反馈 | 误报率 < 20%（用户认可） |
| 知乎内容能带来自然注册 | 发布3篇，追踪来源注册量 | 每篇带来 ≥3 注册 |
| 用户愿意在检查后导出文档 | 监控导出功能使用率 | 使用过 AI 检查的用户中 ≥30% 使用导出 |

### 暂时不做的事项（防止战略漂移）

- **不做手机 App**：Web 版功能未稳定前，移动端是分散精力的陷阱
- **不接纯外包**：帮人改稿、人工排版、PPT 制作均拒绝，无论报价多高
- **不接机构年框合同**（Q2 前）：销售周期 3–6 个月，团队精力不够
- **不做多语言版本**：英文学术市场有 Grammarly，中文是护城河，守住它
- **不搭建自有 LLM**：DeepSeek API 成本已极低，自建算力 ROI 极差
- **不做浏览器插件**（Q3 前）：插件分发和更新成本高，核心场景在文档编辑器
- **不优化 pywebview 桌面版**：已放弃，精力集中在 Web 版
- **不在付费用户 < 50 人时做大规模营销**：先验证留存，再花钱获客
